## v2 — Local UI

User runs:

```bash
maiker run ./app --ui
```

or

```bash
maiker ui
```

### Recommended panels

#### Left panel
- run list
- stage list
- current status
- current agent

#### Center panel
- live log stream
- current action summary
- current prompt summary
- touched files

#### Right panel
- screenshots
- issues
- diffs
- validator reports

#### Bottom bar
- pause
- resume
- add context
- request replan
- approve
- reject

## Interaction rule

UI must not mutate hidden state directly.
All user actions should become explicit workflow events:
- pause_requested
- context_added
- replan_requested
- approval_submitted
